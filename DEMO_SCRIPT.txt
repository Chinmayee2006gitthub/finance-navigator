[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the architecture flow.
First, every user query starts at the Security Checkpoint node. This checkpoint scrubs credit card PII, flags injection threats, and logs actions into an audit trail.
If secure, the query moves to the Orchestrator Agent. The orchestrator coordinates our specialized specialists: the Categorizer Agent, which maps transactions to budgets, and the Savings Agent, which analyses goals.
Both sub-agents query a local MCP Server for secure data fetching.
Next, the Orchestrator passes the aggregated report to the Decision Node. If the report highlights large expenditures or budget violations, it routes to the Human Approval Node where a Human-in-the-Loop pause occurs.
Finally, the results are formatted and displayed to the user in the Finalize Output Node.

Meanwhile, our decoupled local MCP server allows us to fetch transactions, budgets, and savings goals securely without exposing raw database credentials to the model.
